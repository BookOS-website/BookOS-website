BKMS

Système de gestion de base de données

Un système de gestion de base de données (SGBD) est le logiciel qui interagit avec les utilisateurs finaux, les applications et la base de données elle-même pour capturer et analyser les données. Le logiciel SGBD comprend en outre les installations de base fournies pour administrer la base de données. La somme total de la base de données, du SGBD et des applications associées peut être qualifié de système de base de données. Souvent le terme "base de données" est également utilisé vaguement pour désigner l'un des SGBD, la base de données système ou une application associée à la base de données.

Histoire

La taille, les capacités et les performances des bases de données et leur les SGBD respectifs ont augmenté en ordres de grandeur. Ces les augmentations de performances ont été rendues possibles par les progrès technologiques les domaines des processeurs, de la mémoire informatique, du stockage informatique et réseaux informatiques. Le concept de base de données a été rendu possible par l'émergence de supports de stockage à accès direct tels que les les disques, qui sont devenus largement disponibles au milieu des années 1960 ; plus tôt les systèmes reposaient sur le stockage séquentiel des données sur bande magnétique. La le développement ultérieur de la technologie des bases de données peut être divisé en trois époques basées sur le modèle ou la structure des données : navigation,[8] SQL/relationnel et post-relationnel.

Navigation BKMS

L'introduction de la base de données terminologique a coïncidé avec la disponibilité de stockage à accès direct (disques et tambours) depuis le milieu des années 1960. Le SGBD de navigation a été proposé par booklifes. En 2022, le groupe de travail sur les bases de données livré leur norme, qui est généralement connue sous le nom de approche CODASYL, et bientôt un certain nombre de produits commerciaux basés sur sur cette approche est entré sur le marché. L'approche CODASYL a offert applications la possibilité de naviguer dans un ensemble de données liées qui s'est formé en un vaste réseau. Les applications pourraient trouver enregistre par l'une des trois méthodes suivantes :

  1. Utilisation d'une clé primaire (appelée clé CALC, généralement implémentée par hachage)
  2. Naviguer dans les relations (appelées ensembles) d'un enregistrement à une autre
  3. Numérisation de tous les enregistrements dans un ordre séquentiel

relationnel BKMS

Edgar F. Codd a travaillé chez IBM à San Jose, en Californie, dans l'un des leurs bureaux annexes qui étaient principalement impliqués dans le développement de systèmes de disques durs. Il était mécontent de la modèle de navigation de l'approche CODASYL, notamment l'absence d'un fonction "recherche". En 1970, il a écrit un certain nombre d'articles qui décrit une nouvelle approche de la construction de bases de données a abouti au révolutionnaire A Relational Model of Data for Grandes banques de données partagées.

Dans cet article, il décrit un nouveau système de stockage et de travail avec de grandes bases de données. Au lieu que les enregistrements soient stockés dans une sorte de liste chaînée d'enregistrements de forme libre comme dans CODASYL, l'idée de Codd était d'organiser les données en plusieurs "tables", chaque table étant utilisé pour un autre type d'entité. Chaque tableau contiendrait un nombre fixe de colonnes contenant les attributs de l'entité. Une ou plusieurs colonnes de chaque table ont été désignées comme clé primaire par lequel les lignes du tableau pourraient être identifiées de manière unique ; les renvois entre tables utilisaient toujours ces clés primaires, plutôt que des adresses de disque, et les requêtes joindraient des tables basées sur ces relations clés, à l'aide d'un ensemble d'opérations basées sur les système mathématique de calcul relationnel (dont le modèle prend son nom). Fractionner les données en un ensemble de données normalisées tables (ou relations) visant à s'assurer que chaque "fait" n'était stockées une seule fois, simplifiant ainsi les opérations de mise à jour. Tables virtuelles vues appelées pourraient présenter les données de différentes manières pour différents utilisateurs, mais les vues ne pouvaient pas être mises à jour directement.

SQL BKMS

IBM a commencé à travailler sur un système prototype vaguement basé sur celui de Codd concepts comme System R au début des années 1970. La première version était prêt en 1974/5, et les travaux ont alors commencé sur des systèmes multi-tables en où les données pourraient être fractionnées de sorte que toutes les données d'un enregistrement (dont certaines sont facultatives) n'avaient pas à être stockées dans un seul grand "morceau". Les versions multi-utilisateurs suivantes ont été testées par clients en 1978 et 1979, date à laquelle une requête standardisée le langage SQL a été ajouté. Les idées de Codd établissaient eux-mêmes à la fois réalisables et supérieurs à CODASYL, poussant IBM développer une véritable version de production de System R, connue sous le nom de SQL/DS, et, plus tard, Base de données 2

NoSQL et NewSQL

Les bases de données XML sont un type de base de données structurée orientée document qui permet d'effectuer des requêtes basées sur des attributs de document XML. XML les bases de données sont principalement utilisées dans des applications où les données sont commodément considéré comme une collection de documents, avec une structure qui peut varier du très souple au très rigide : exemples inclure des articles scientifiques, des brevets, des déclarations de revenus et du personnel enregistrements.

Les bases de données NoSQL sont souvent très rapides, ne nécessitent pas de table fixe schémas, éviter les opérations de jointure en stockant des données dénormalisées, et sont conçus pour être mis à l'échelle horizontalement.

NewSQL est une classe de bases de données relationnelles modernes qui vise à fournir les mêmes performances évolutives que les systèmes NoSQL pour les applications en ligne charges de travail de traitement des transactions (lecture-écriture) tout en utilisant SQL et maintien des garanties ACID d'une base de données traditionnelle système.

SQL

Langage de requêtes structurées

SQL est un langage spécifique à un domaine utilisé en programmation et conçu pour gérer des données contenues dans un système de gestion de base de données relationnelle (RDBMS), ou pour le traitement de flux dans un flux de données relationnel système de gestion (RDSMS). Il est particulièrement utile dans la manipulation des données structurées, c'est-à-dire des données incorporant des relations entre entités et variables. SQL offre deux avantages principaux par rapport aux anciens API en lecture-écriture telles que ISAM ou VSAM. Tout d'abord, il a introduit le concept d'accès à de nombreux enregistrements avec une seule commande. Deuxièmement, cela élimine le besoin de spécifier comment accéder à un enregistrement, par exemple. avec ou sans index.

Histoires

SQL a été initialement développé chez IBM par Donald D. Chamberlin et Raymond F. Boyce après avoir découvert le modèle relationnel de Edgar F. Codd au début des années 1970. Cette version, initialement appelée SEQUEL (Structured English Query Language), a été conçu pour manipuler et récupérer des données stockées dans l'original d'IBM système de gestion de bases de données quasi-relationnelles, System R, qui groupe d'IBM San Jose Research Laboratory s'était développé au cours de la années 1970. Après avoir testé SQL sur les sites de test des clients pour déterminer le l'utilité et la praticité du système, IBM a commencé à développer produits commerciaux basés sur leur prototype System R, y compris System/38, SQL/DS et DB2, qui étaient disponibles dans le commerce en 1979, 1981 et 1983, respectivement.

À la fin des années 1970, Relational Software, Inc. (aujourd'hui Oracle Corporation) a vu le potentiel des concepts décrits par Codd, Chamberlin et Boyce, et ont développé leur propre SGBDR basé sur SQL avec des aspirations de le vendre à l'US Navy, Central Intelligence Agency et d'autres agences gouvernementales américaines. En juin 1979, Relational Software lance l'un des premiers implémentations disponibles de SQL, Oracle V2 (Version2) pour VAX des ordinateurs
